# QR-API no v0 (Vercel)

O v0 gera componentes React. A chave da QR-API jamais pode ir para o client-side.
O padrao correto e: componente React chama uma API Route do Next.js que fica no servidor.

## IMPORTANTE: seguranca da chave

Nunca use `NEXT_PUBLIC_QR_API_KEY`. Variaveis com `NEXT_PUBLIC_` sao expostas no bundle
do browser. A chave `QR_API_KEY` (sem o prefixo NEXT_PUBLIC_) fica apenas no servidor.

## Route Handler (Next.js App Router)

Cole o prompt abaixo no v0:

```
Crie um route handler Next.js em app/api/whatsapp/route.ts que:
1. Aceita POST com { phone: string, message: string }
2. Busca QR_API_KEY e QR_INSTANCE_ID das variaveis de ambiente do servidor
3. Chama a QR-API para enviar uma mensagem de texto
4. Retorna { messageId, status } ou { error, code } com status HTTP correto
```

Codigo gerado esperado:

```typescript
// app/api/whatsapp/route.ts
import { NextRequest, NextResponse } from "next/server";

export async function POST(req: NextRequest) {
  const { phone, message } = await req.json();

  if (!phone || !message) {
    return NextResponse.json(
      { error: "phone e message sao obrigatorios" },
      { status: 400 }
    );
  }

  const apiKey = process.env.QR_API_KEY;
  const instanceId = process.env.QR_INSTANCE_ID;

  if (!apiKey || !instanceId) {
    console.error("QR_API_KEY ou QR_INSTANCE_ID nao configuradas");
    return NextResponse.json({ error: "configuracao ausente" }, { status: 500 });
  }

  const res = await fetch("https://api.qrapi.flipt.com.br/v1/messages", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${apiKey}`,
      "Content-Type": "application/json",
      "Idempotency-Key": crypto.randomUUID(),
    },
    body: JSON.stringify({
      instanceId,
      to: phone,
      type: "text",
      text: message,
    }),
  });

  const data = await res.json();

  if (!res.ok) {
    console.error("QR-API error:", data.error?.code, data.request_id);
    return NextResponse.json(
      { error: data.error?.message, code: data.error?.code, request_id: data.request_id },
      { status: res.status }
    );
  }

  return NextResponse.json({ messageId: data.id, status: data.status });
}
```

## Componente React que usa a route

```tsx
// components/EnviarWhatsapp.tsx
"use client";

import { useState } from "react";

export function EnviarWhatsapp() {
  const [phone, setPhone] = useState("");
  const [message, setMessage] = useState("");
  const [status, setStatus] = useState<string | null>(null);
  const [loading, setLoading] = useState(false);

  async function handleSubmit(e: React.FormEvent) {
    e.preventDefault();
    setLoading(true);
    setStatus(null);

    try {
      const res = await fetch("/api/whatsapp", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ phone, message }),
      });

      const data = await res.json();

      if (!res.ok) {
        setStatus(`Erro: ${data.error || "falha ao enviar"}`);
      } else {
        setStatus(`Enviado. ID: ${data.messageId}`);
      }
    } catch (err) {
      setStatus("Erro de conexao");
    } finally {
      setLoading(false);
    }
  }

  return (
    <form onSubmit={handleSubmit} className="flex flex-col gap-4 max-w-md">
      <input
        type="tel"
        placeholder="Numero (5511999998888)"
        value={phone}
        onChange={(e) => setPhone(e.target.value)}
        className="border rounded p-2"
      />
      <textarea
        placeholder="Mensagem"
        value={message}
        onChange={(e) => setMessage(e.target.value)}
        className="border rounded p-2"
        rows={3}
      />
      <button type="submit" disabled={loading} className="bg-green-600 text-white rounded p-2">
        {loading ? "Enviando..." : "Enviar WhatsApp"}
      </button>
      {status && <p className="text-sm text-gray-600">{status}</p>}
    </form>
  );
}
```

## Variaveis de ambiente no Vercel

No painel Vercel, va em Settings > Environment Variables e adicione:
- `QR_API_KEY` = sk_live_xxx (escopo: Production, Preview, Development)
- `QR_INSTANCE_ID` = inst_xxx

Localmente, crie `.env.local` (nao commitado):
```
QR_API_KEY=sk_test_xxx
QR_INSTANCE_ID=inst_xxx
```
